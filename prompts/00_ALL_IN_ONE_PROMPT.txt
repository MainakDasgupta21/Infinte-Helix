PROJECT: Infinite Helix — AI-Powered Micro-Wellness & Productivity Assistant
         for Women Employees
================================================================================

Build a complete full-stack web application called "Infinite Helix". It is
a background AI assistant that helps women employees manage work, emotions,
energy, and mental well-being through context-aware wellness nudges, emotion AI,
an LLM-powered chatbot, and optional cycle-aware insights — all with a
beautiful dark-first UI.

The project must support both a zero-config demo mode (SQLite + mock AI) and
a full production mode (Firebase + HuggingFace models + Groq LLM).

Build it step-by-step, file-by-file, following the instructions below exactly.

##  SECTION 1 — PROJECT OVERVIEW & FEATURES                                   ##
##                                                                            ##
################################################################################

PROJECT NAME: Infinite Helix
TAGLINE: "AI Micro Wellness Assistant"
TARGET AUDIENCE: Women employees in office/remote work environments

VISION & GOALS:
1. Supportive, non-intrusive wellness nudges during the workday (hydration,
   stretch, eye rest, breathing exercises).
2. Track work behavior (keyboard/mouse activity, screen time, idle periods)
   to detect fatigue and stress patterns.
3. Emotion and sentiment analysis on journal entries using AI, with supportive
   reframing for negative sentiments.
4. Optional, private menstrual cycle tracker adjusting energy forecasts and
   nudge tone based on cycle phase.
5. Google Calendar and Microsoft Graph integration to detect upcoming meetings
   and suggest pre-meeting calm exercises.
6. Conversational AI chatbot ("Helix") that is page-context-aware and delivers
   personalized wellness advice via Groq LLM (llama-3.3-70b-versatile).
7. Weekly wellness report cards with printable/downloadable reports.
8. All cycle/pregnancy data stored strictly on-device (localStorage) for privacy.
9. Support both a full Firebase cloud mode and a zero-config demo mode with
   SQLite fallback and mock AI models.

CORE FEATURES (15 total):

1. WORK BEHAVIOR TRACKING — pynput keyboard/mouse monitoring, idle detection
   (60s threshold), continuous work monitoring (120 min fatigue threshold),
   typing intensity, psutil screen time tracking with app categorization.

2. CONTEXT-AWARE WELLNESS NUDGES — hydration, stretch, eye rest, breathing
   reminders with cooldown (30 min production / 5 min demo), priority-based
   evaluation, dismissible feed on dashboard.

3. EMOTION & SENTIMENT AI — HuggingFace emotion
   (j-hartmann/emotion-english-distilroberta-base, 7 classes: joy, sadness,
   anger, fear, surprise, disgust, neutral) + sentiment
   (cardiffnlp/twitter-roberta-base-sentiment, 3 classes). Supportive
   reframing for negative entries. Demo mode: keyword-based mock classifiers.

4. EMOTION JOURNAL — write entries, instant AI analysis with confidence
   scores, entry history with emotion badges, AI-generated supportive responses.

5. CYCLE ENERGY MODE (Optional, Private) — 28-day cycle model with 4 phases:
   Days 1-5 Menstrual (rest-focused), Days 6-13 Follicular (energy rising),
   Days 14-16 Ovulatory (peak energy), Days 17-28 Luteal (winding down).
   Cycle ring SVG donut, energy forecast, mood/flow/symptom logging.
   ALL data stored in localStorage ONLY — never sent to server.

6. PREGNANCY / MOTHERHOOD SHIELD MODE — toggle between cycle and pregnancy,
   due date with trimester calculation, week-by-week tracking, appointment
   countdown, adapted deep pelvic breathing exercises.

7. PRE-MEETING CALM — Google Calendar + Microsoft Graph integration,
   detects meetings, suggests breathing/confidence exercises,
   ConfidenceBreath component with guided breathing animation.

8. AI CHATBOT ("Helix") — Groq API (llama-3.3-70b-versatile), page-context
   awareness, 20 intent categories via regex, template fallback without key,
   session management, message history, emotional trajectory tracking,
   quick reply suggestions, crisis/distress detection with helpline numbers.

9. DASHBOARD — real-time wellness score (0-100), productivity score with
   streak, screen time donut chart, break balance, focus timeline, hydration
   tracker (250ml/glass), self-care tracker, today's tasks widget, nudge feed,
   AI Wellness Engine status indicator.

10. REPORTS & ANALYTICS — 3-tab Reports page:
    Tab 1: Digital Body Language (typing patterns, stress events timeline)
    Tab 2: Daily Wellness Report (metrics, mental load, printable HTML)
    Tab 3: Weekly Report Card (charts via Chart.js, printable HTML with
    inline SVG charts). Manager-safe vs private report variants.

11. TODO / TASK MANAGEMENT — tasks with categories (work/personal/health/
    meeting), reminders, today/upcoming/history views, date navigation,
    overdue highlighting.

12. SETTINGS — meal reminder configuration, break intervals, notification
    preferences, theme toggle (dark/light).

13. AUTHENTICATION — Firebase Auth (email/password + Google OAuth 2.0),
    Bearer token on all API requests. Demo mode: local user ID generation.

14. MULTI-CHANNEL NOTIFICATIONS — plyer desktop toasts, browser Web
    Notifications API, in-app notification overlay, stress intervention
    toast (full-screen calming), break reminder toast.

15. STRESS DETECTION — standalone stress_agent.py using pynput, tracks
    typing patterns (speed, delete ratio, burst detection), posts to
    /api/stress/ingest every 10s, frontend polls /stress/latest,
    stress intervention overlay with breathing exercise.

DESIGN PHILOSOPHY:
- Supportive, not surveillance — nudges feel caring, not intrusive
- Privacy-first — cycle/pregnancy data stays on-device (localStorage)
- Inclusivity — cycle tracking is entirely optional and toggleable
- Beautiful UI — dark-first bento-grid layout, glass morphism cards,
  smooth Framer Motion animations, Plus Jakarta Sans typeface
- Zero-config demo — works immediately without Firebase, Groq, or ML models
- Women-centric but not exclusionary

    FATIGUE_THRESHOLD_MINUTES (int, 120), DESKTOP_NOTIFICATIONS (bool, True).

FILE: backend/app/__init__.py — FLASK APP FACTORY
  create_app() function that:
  1. Creates Flask app, loads Config, configures CORS for /api/*
  2. Adds after_request handler for CORS headers
  3. Imports and registers 17 blueprints (see directory tree for prefixes)
  4. Registers GET /api/health endpoint returning
     {"status":"ok","service":"Infinite Helix API","demo_mode":<bool>}
  5. Calls init_firebase() from firebase_service
  6. Calls _init_ai_models(app)

  Module-level: activity_monitor = None (set by run.py)

  _init_ai_models(app):
    If DEMO_MODE: use _DemoEmotionDetector and _DemoSentimentAnalyzer
    Else: try loading real HuggingFace models, fallback to demo on error

  _DemoEmotionDetector class (in same file):
    - _KEYWORDS dict mapping mood -> keyword lists:
      stress: [stress, overwhelm, tired, exhaust, burnout, deadline, ...]
      happy: [happy, great, wonderful, excited, love, amazing, ...]
      angry: [angry, frustrated, annoyed, furious, unfair, hate, ...]
      fear: [scared, anxious, worried, nervous, afraid, panic, ...]
    - _PROFILES dict: each mood -> {emotion, confidence, all_emotions}
    - analyze(text): lowercase, match keywords, return profile or neutral

  _DemoSentimentAnalyzer class (in same file):
    - _NEGATIVE keywords, _POSITIVE keywords
    - _REFRAMES list: 7 supportive reframing messages
    - analyze(text): check keywords, return {sentiment, confidence,
      all_sentiments, reframe} — rotate reframe for negative

FILE: backend/app/middleware.py
  require_auth decorator:
    - Extract Bearer token from Authorization header
    - Verify via Firebase Admin SDK
    - On success: set request.uid, request.email
    - On failure + DEMO_MODE: set request.uid = 'demo-user-001'
    - On failure + not demo: return 401

FILE: backend/run.py — ENTRY POINT
  - Load .env, insert backend dir into sys.path
  - start_background_tracker(): create ActivityMonitor in daemon thread,
    store in app_module.activity_monitor
  - main(): create_app(), start tracker, run on 0.0.0.0:5000 with
    use_reloader=False

FILE: backend/stress_agent.py — STANDALONE STRESS DETECTOR
  - pynput keyboard listener tracking keystrokes in a deque (last 30s)
  - Every 10s: compute {keystrokes, backspaces, speed} and POST to
    http://localhost:5000/api/stress/ingest
  - Privacy: counts keys only, never records content

##                                                                            ##
##  SECTION 5 — BACKEND ROUTES (17 Blueprints)                                ##
##                                                                            ##
################################################################################

Every route requiring auth uses require_auth decorator. In demo mode,
any Bearer token is accepted and assigned a demo user ID.

1. AUTH — auth_bp (/api/auth)
   POST /register: verify token, save user doc. POST /sync: update last_login.
   GET /profile: return user doc.

2. EMOTION — emotion_bp (/api/emotion)
   POST /analyze: {text} -> emotion_detector.analyze + sentiment_analyzer.analyze
   -> {emotion, confidence, all_emotions, sentiment, reframe, all_sentiments}

3. SENTIMENT — sentiment_bp (/api/sentiment)
   POST /analyze: {text} -> {sentiment, confidence, all_sentiments, reframe}

4. JOURNAL — journal_bp (/api/journal)
   POST /: analyze text + save entry + save mood log.
   GET /: list entries. GET /<id>: get entry.

5. DASHBOARD — dashboard_bp (/api/dashboard)
   GET /today: comprehensive metrics — wellness score (weighted: hydration 30%,
   breaks 30%, self-care 20%, activity 20%), screen time, activity stats,
   hydration, self-care, mood, streak, focus sessions.
   GET /screen-history: screen time history for N days.

6. REPORTS — reports_bp (/api/reports)
   GET /weekly: weekly report via wellness_report_service.

7. TRACKER — tracker_bp (/api/tracker)
   GET /status, POST /start, POST /stop — activity monitor control.

8. NUDGE — nudge_bp (/api/nudge)
   POST /generate: {context} -> nudge. GET /pending: undismissed nudges.
   POST /<id>/dismiss.

9. CALENDAR — calendar_bp (/api/calendar)
   GET /meetings, GET /next, GET /status, GET /authorize?provider=,
   GET /google/callback, GET /callback (Microsoft), POST /disconnect,
   POST /events. Demo mode returns mock meeting data.

10. CYCLE — cycle_bp (/api/cycle)
    GET /suggestions/<phase>: phase-specific wellness suggestions.
    POST /phase: set current phase.

11. HYDRATION — hydration_bp (/api/hydration)
    POST /log: {amount_ml?} default 250. GET /today: {ml_today, entries}.

12. SELF-CARE — selfcare_bp (/api/selfcare)
    POST /log: {action}. GET /today: {stretch, eye_rest}.

13. PRIVATE CARE — privatecare_bp (/api/privatecare)
    POST /log: {type, note?}. GET /history. GET /period-history.
