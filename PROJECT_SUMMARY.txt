1. PROJECT OVERVIEW
--------------------------------------------------------------------------------
Infinite Helix is a full-stack AI-powered desktop/web application designed to
help women employees manage work, emotions, energy, and mental well-being
throughout the workday. It delivers context-aware wellness nudges, emotion AI
analysis, cycle-aware insights, and productivity tracking — all while keeping
sensitive features optional and private.

The project was built by Team Infinite Helix for a hackathon, with a focus on
women-centric wellness while maintaining inclusivity (e.g., cycle tracking is
optional and private).

4. KEY FEATURES
--------------------------------------------------------------------------------
4.1  Work Behavior Tracking
     - Keyboard and mouse activity monitoring via pynput
     - Idle detection and continuous work duration tracking via psutil
     - Screen time measurement and history

4.2  Context-Aware Wellness Nudges
     - Smart reminders for hydration, stretching, eye rest, and breathing
     - Designed to feel supportive rather than intrusive
     - Nudge generation, pending queue, and dismissal workflow
     - Adjustable cooldown intervals (reduced in demo mode)

4.3  Emotion & Sentiment AI
     - HuggingFace Transformers models for emotion detection (7 classes)
       and sentiment analysis
     - Confidence scores for each analysis
     - Supportive reframing suggestions when stress is detected
     - Falls back to keyword-based detection in demo mode

4.4  Cycle Energy Mode (Private Care)
     - Optional menstrual phase logging — fully toggleable and private
     - Phase-specific wellness suggestions (energy-level-aware nudges)
     - Period history tracking
     - Adjusts recommendations based on current cycle phase

4.5  Calendar & Pre-Meeting Calm
     - Google Calendar API and Microsoft Graph/Outlook integration
     - OAuth2 flows for both providers
     - Meeting detection and pre-meeting breathing/calm suggestions
     - Demo mode provides mock meeting data

4.6  Chatbot "Helix"
     - AI-powered conversational assistant
     - Uses Groq API for LLM-based responses when API key is available
     - Falls back to template/rule-based replies otherwise
     - Context-aware (knows current page, user state)
     - Chat history management and quick reply suggestions
     - Safety boundaries and intent classification

4.7  Multi-Channel Notifications
     - Desktop toasts via plyer (cross-platform)
     - Browser Web Notifications API
     - In-app notification overlay
     - Meal, eye rest, and private care reminders with scheduling

4.8  Dashboard & Reports
     - Real-time wellness score aggregation
     - Screen time charts and stress heatmaps
     - Weekly wellness report cards
     - Hydration and self-care tracking visualizations

4.9  Emotion Journal
     - Write free-form journal entries
     - Instant AI emotion and sentiment analysis on each entry
     - Entry history with browsing

4.10 Todo / Task Management
     - Create, view, toggle, and delete daily tasks
     - Today's task view integrated into dashboard

4.11 User Settings & Theming
     - Light and dark mode theme toggle
     - User profile management
     - Configurable settings via API

6. AUTHENTICATION & SECURITY
--------------------------------------------------------------------------------
- Client-side: Firebase Authentication with email/password and Google OAuth 2.0
- Server-side: Firebase Admin SDK verifies Bearer tokens from request headers
- Middleware (require_auth) sets request.uid and request.email on each request
- Demo Mode: bypasses token validation, assigns a fixed demo user ID
- Calendar OAuth: uses state parameter to associate provider tokens with
  Firebase user IDs during redirect flows
- API client (axios) automatically attaches Firebase ID tokens to all requests


7. DATABASE & STORAGE
--------------------------------------------------------------------------------
Primary:  Firebase Cloud Firestore
          - Used when valid Firebase credentials are configured
          - Stores all user data, journal entries, settings, chat history, etc.

Fallback: SQLite (local.db)
          - Generic "documents" table storing collection name + JSON blobs
          - Activates when Firestore is unavailable or write operations fail
          - Enables fully offline / no-credential demo operation


8. AI & MACHINE LEARNING
--------------------------------------------------------------------------------
Emotion Detection:
  - Model: DistilRoBERTa-based emotion classifier (HuggingFace)
  - 7 emotion classes
  - Demo mode: keyword-based fallback (no model download needed)

Sentiment Analysis:
  - Model: RoBERTa-based sentiment analyzer (HuggingFace)
  - Positive/negative/neutral classification with confidence
  - Demo mode: keyword-based fallback

Chatbot (Helix):
  - Primary: Groq API for LLM chat completions
  - Fallback: Template and rule-based response system
  - Persona-driven system prompt with safety boundaries
  - Context-aware (current page, user wellness state)

Nudge Engine:
  - Generates context-aware wellness recommendations
  - Factors: screen time, activity patterns, hydration, break history


9. DEMO MODE
--------------------------------------------------------------------------------
Setting DEMO_MODE=true in the backend .env file enables:
  - Keyword-based emotion/sentiment detection (no model downloads)
  - SQLite/in-memory storage fallback (no Firebase credentials required)
  - Realistic mock data for calendar meetings and reports
  - Reduced nudge cooldown (5 minutes) for demo visibility
  - Desktop notifications via plyer
  - Fixed demo user authentication bypass


10. HOW TO RUN
--------------------------------------------------------------------------------
Backend:
  cd Infinite_helix/backend
  python setup_demo.py          # Creates .env, installs dependencies
  python run.py                 # Starts Flask API on port 5000

Frontend (separate terminal):
  cd Infinite_helix/frontend
  npm install
  npm run dev                   # Starts React dev server on port 3000

Open http://localhost:3000 — register or sign in to begin.

For environment variable configuration, see backend/.env.example.
